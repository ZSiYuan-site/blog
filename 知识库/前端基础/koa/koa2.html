    # koa2中间件

    什么是中间件

    1.一个流程上，独立的业务模块
2.可扩展，可插拔
3.类似于工厂的流水线
    1
    2
    3
    4

    为什么使用中间件

    1.拆分业务模块，使代码清晰
2.统一使用中间件，使得各业务代码都规范标准
3.扩展性好，易添加，易删除
    1
    2
    3
    4

    一个http请求流程 :

    req--> bodyparser中间件 --> json中间件 ---> logger中间件 ---> static中间件 -->views中间件 -->自定义log---> routes中间件 ---> res
    1
    2

    注意：其实routes也是中间件，只不过要按照router的规则走

    # koa2洋葱圈模型

    中间件机制，是koa2的精髓
每个中间件都是async函数
中间件的运行机制，就像洋葱圈
    1
    2
    3
    4

    备注 ：注册中间件的时候，并不会调用中间件，需要访问的时候才会执行中间件

    代码演示 中间件的洋葱圈

    const Koa = require('koa')

const app = new Koa()

// logger
app.use(async (ctx, next) => {
    await next() // 执行下一个中间件  
    const rt = ctx.response.get('X-Response-Time')
    console.log(`${ctx.method}---${ctx.url}----${rt}`)
})

// x-response-time 
app.use(async (ctx, next) => {
    const start = Date.now()  
    await next()   // 执行下一个中间件 
    const ms = Date.now() - start   // 计算时间差 
    ctx.set('X-Response-Time',`${ms} ms`)   // 记录时间差,这里的ms不能写成中的【毫秒】
})

// response
app.use(async (ctx, next) => {
    ctx.body = "hello world"
})

app.listen(3001)
console.log('koa2 已经监听3001端口')

    # cookie是啥

    存储在浏览器的一段字符串，最大5kb
每个域都有一个cookie，跨域不共享
格式如 k1=v1;k2=v2; (可结构化)
    1
    2
    3

    备注 ：

    每次http请求，cookie都会随着传递到服务端
服务端可以修改cookie 再返回给前端
默认，跨域不可以传递cookie

    # 使用node原生服务端操作 cookie

    步骤 ：设置cookie 获取cookie 结构化cookie

    const http = require('http')

const server = http.createServer((req, res) => {
    // 设置cookie
    res.setHeader('Set-Cookie', 'a=400')
    res.end('cookie test')

    // 获取cookie 
    const cookieStr = req.headers.cookie
    console.log('cookie is ', cookieStr)

    // 结构化 cookie
    // 假如 cookieStr 为 'a=100; b=200' ---转换为 --》 {a:100, b:200}
    const cookieObg = {}
    cookieStr.split(';').forEach(cookieItemStr => {
        const arr = cookieItemStr.trim().split('=')
        const key = arr[0] //'a'
        const val = arr[1] // '100'
        cookieObg[key] = val
    })
    console.log('cookie obj is ', cookieObg)
})

server.listen(3002)
console.log('监听3002 端口成功')

前端再次请求其他接口，就会带着上述的cookie
服务端判断对应的cookie有无，进行验证

    # koa2实现登录

    有两种情况：

    1。直接请求 /login ，也就是直接去登录,配合使用 sequelize ,创建User模型等操作。。。

    在routes/users.js :

    router.get('/login', async (ctx, next) => {
const {
    username,
    password
} = ctx.query

const user = await User.findOne({
    where: {
    userName: username,
    password
    }
})
if (user != null) {
    // 登录成功，设置session
    ctx.session.userInfo = user
    // 返回
    ctx.body = {
    status: 200,
    data: user
    }
    return   //注意一定要 return
}

// 登录失败
ctx.body = {
    staus: -1,
    message: '用户名字或者密码错误'
}
})

    2.请求 商品列表，/list 是需要登录成功,才能请求成功, 这时候就需要定义登录中间件

    在middlewares/loginCheck.js :

    // 登录验证的中间件
async function loginCheck(ctx, next) {
    const userInfo = ctx.session.userInfo
    if (userInfo && userInfo.userName) {
        // 登录验证成功
        await next()
        return
    }
    // 登录失败
    ctx.body = {
        errno: -1,
        message: '请登录'
    }
}
module.exports = loginCheck
