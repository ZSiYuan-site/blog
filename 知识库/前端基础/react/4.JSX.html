    # 什么是JSX

    • 什么是JSX: 是一种JS和HTML混合的语法,将组件的结构、数据甚至样式都聚合在一起定义组件
    • 比如如下：
    ReactDOM.render(
  <h1>Hello</h1>,
  document.getElementById('root')
);

    # 什么是元素

    • JSX其实只是一种语法糖,最终会通过babeljs转译成createElement语法
    • React元素是构成React应用的最小单位
    • React元素用来描述你在屏幕上看到的内容
    • React元素事实上是普通的JS对象,ReactDOM来确保浏览器中的DOM数据和React元素保持一致 比如如下代码：
    // 下面一整陀就叫 “react元素”
<h1 className="title" style={{color:'red'}}>hello</h1>

let element =  React.createElement("h1", {

    # 更新元素渲染

    • React 元素都是immutable不可变的。当元素被创建之后，你是无法改变其内容或属性的。一个元素就好像是动画里的一帧，它代表应用界面在某一时间点的样子
    • 更新界面的唯一办法是创建一个新的元素，然后将它传入ReactDOM.render()方法
    import React from 'react';
import ReactDOM from 'react-dom';
let root = document.getElementById('root');
function tick() {
    // 只能每次【创建一个新的元素】进行渲染
    const element = (
        <div>
            <span>当前时间：</span>
            {new Date().toLocaleTimeString()}
        </div>
    );
    ReactDOM.render(element, root);
}
setInterval(tick, 1000);

    # React只会更新必要的部分

    • 就比如上面的例子中，只会更新时间，不会更新文本
    • 在渲染的时候，会进行比较老的虚拟DOM和新的虚拟DOM的区别，然后只更新发送了变化的部分，这就是所谓了的dom-diff
    • 即便我们每秒都创建了一个描述整个UI树的新元素，React DOM 也只会更新渲染文本节点中发生变化的内容
    <body>
    <div id="root"></div>

    <!--引入react核心库-->
    <script type="text/javascript" src="./js/react.development.js"></script>
    <!--引入操作react操作DOM库-->
    <script type="text/javascript" src="./js/react-dom.development.js"></script>
    <!--引入babel-->
    <script type="text/javascript" src="./js/babel.min.js"></script>

    <script type="text/babel">
        function tick() {
            const element = (
                <div>
                    <h1>Hello, world!</h1>
                    <h2>It is {new Date().toLocaleTimeString()}.</h2>
                </div>
            );
            ReactDOM.render(element, document.getElementById('root'));
        }

        setInterval(tick, 1000);
    </script>

</body>

    # JSX的执行过程（重点）

    • 1.我们写代码的时候写的JSX，如 <h1>hello</h1>
    • 打包的时候，会调用webpak中的babel-loader把JSX写法转成JS的写法，即createElement
    • 打开浏览器，浏览器会执行createElement方法(返回值是虚拟DOM)，所以得到虚拟DOM,也就是React元素，它是一个普通的JS对象，描述了你在界面上想看的DOM元素的样子
    • 把React元素（虚拟DOM）给了ReactDOM.render方法，该方法会把虚拟DOM转成真实DOM，并且插入到页面中

    # fiber

    • 就是把这整个任务拆分成一个个的小任务，全部放在浏览器空闲的时间执行，这样不会阻塞浏览器高优先级的任务

    # 可变与不可变

    • immutable表示不可变数据，mutable表示可变数据，react要求数据是不可变的，数据一旦创建就不能修改

    # JSX使用示例

    <!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>JSX练习</title>
</head>

  <div id="test"></div>
 
  <!--引入react核心库-->

  <script type="text/javascript" src="../js/react-dom.development.js"></script>

  <script type="text/javascript" src="../js/babel.min.js"></script>

  <script type="text/babel">
  //准备一些数据
  let data = ['Angular','React','vue','jQuery'] //数据的数组
  //let data2 = [<li>Angular</li>,<li>React</li>,<li>vue</li>,<li>jQuery</li>] //标签数组

  //1.创建虚拟DOM
  let vDOM = (
    <div>
        <h2>前端js框架列表</h2>  
        <ul>
          {/* 把数据的数组加工成标签的数组 */}
          {
            data.map((item,index)=>{
              /* 我们暂时使用index作为每一个child的key，但是这种某些时候会产生一些问题，后面会仔细分析 */
              return <li key={index}>{item}</li>
            })
          }
        </ul>
    </div>
  )

  //2.渲染虚拟DOM到页面
  ReactDOM.render(vDOM,document.getElementById('test'))
  
  </script>

</body>
</html>

    # dangerouslySetInnerHTML的使用

    • JSX中传入的值都会自动被HTML转义
    • 这样可以防止XSS攻击
    • 比如如下：
    <div>{"<p></p>"}</div>
// 会被转译为：&lt;p&gt;&lt/p&gt;
    1
    2
    • 如果要输出不转义，可以使用dangerouslySetInnerHTML属性，其实就是相当于调用了DOM元素的innerHTML属性
    <div dangerouslySetInnerHTML={{__html:"<p>你好</p>"}}></div>
    1
