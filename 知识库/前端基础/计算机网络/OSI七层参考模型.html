    OSI七层参考模型

    zsy 2021/12/12

    # 历史背景

    计算机网络分层结构：

    • 7层OSI参考模型，该模型是最早法定标准
    • 4层TCP/IP参考模型，这是由于it市场而诞生的
    • 5层的体系结构，这是由于学习能更好理解而产生的

    前话：

    • 早先年每家大公司都会有自己内部的网络体系结构
    • 但是只有使用该家公司的产品，才能使用其对应的网络体系结构，这样就很不方便
    • 最后，为了支持异构（即不同网络体系结构）的互联互通，国际标准组织（ISO）提出开放系统互连（OSI）参考模型
    • 但是理论成功，市场失败，最终干不过美国国防部的TCP/IP，不过OSI还是有很多值得学习的

    # OSI七层参考模型

    # 应用层

    • 用户与网络的界面，所有能和用户交互【产生网络流量的程序】（比如qq，需要连网才能够使用）
    • 不过记事本，不需要上网就能使用，因此记事本不属于【产生网络流量的程序】，也就不属于应用层的范畴
    • 典型的应用层服务：文件传输（使用的是FTP协议） 、电子邮件（SMTP）、万维网（HTTP）

    # 表示层

    • 用于处理在两个通信系统（手机、电脑）中交换信息（数据传递）的表示方式（通常是对于语法和语义的处理）

    有如下功能：

    • 功能一：数据格式变化（翻译官）
      • 由于不同主机的编码和表示格式都不太一样，但是又需要进行数据交换，因此表示层的数据格式变化功能就体现出来了
      • 比如：可以将比特流 0101010101 转换为 图片。。
    • 功能二：数据加密解密
      • 比如说微信上聊天，可能会提及到支付宝密码，那么会先在发送端进行加密，再放到链路上传输，如果不进行加密，可能会有黑客对此进行抓包，那么哦豁完了，你被盗号了
      • 差不多了对方那儿了，就会对其进行解密
    • 功能三：数据压缩和恢复
      • 比如你和你女朋友在微信聊天，由于视频所涉及到图像非常大，则会先进行压缩，然后再传到链路上进行传输，到了接收端时，再对该被压缩的图像文件解压缩，或者说恢复

    # 会话层

    比如打电话，还没有打通，这时候说话，对方也听不到，即会话层 其实就是先建立好连接，然后呢数据才能传过去。。。

    • 向表示层实体/用户进程提供建立连接并在连接上有序地传输数据
    • 再比如，你打开百度，同时在看电影，此时你快进你的电影，是不会影响到你的百度，即这两个会话是相互独立的

    有如下功能：

    • 功能一：
      • 建立、管理、终止会话
    • 功能二：使用校验点可使会话在通信失效时从 校验点/同步点 继续恢复通信，实现数据同步。
      • 那么啥是校验点/同步点：比如我们要传输一个大文件，那么会把该文件切分成很多段，然后插入多个校验点，如果出现网络中断，会话停止，就不需要重新传
      • 比如你在传一本电子小说，在传到第四章的时候，网络断了，此时又刚好在第四章最近那里找到一个校验点，那么就不需要重头开始传，只要在校验点那个位置开始传就可以了

    # 传输层

    • 负责主机中两个进程的通信，即【端到端】（是指运行在两个不同主机的进程之间的通信）的通信，传输单位是报文段或用户数据报

    功能如下：

    • 功能一：可靠传输、不可靠传输
      • 啥是可靠传输（这是一个基于确认机制的过程）：比如【电脑A---很多路由器----电脑B】，A要发一个pdf文件给B，会先把pdf切分成报文段，然后一个接一个放到链路上进行传输，当接收端每收到一个报文段，都会立刻告诉发送端（俺收到啦，你可以再发下一个了），发送端收到确认信息后，才能继续往下发送，如果突然A没有确认消息，那么A会重新发送这个报文段，直到收到确认消息。。。
      • 啥是不可靠传输：如果你就发了一句，‘在吗’，一个数据报就能解决问题的，就不要有上面那么复杂的流程。。
    • 差错控制：如果发的报文段错了，或者丢了，那么该功能就体现出来了，会对其进行纠正。。
    • 流量控制 : 如果接收方来不及接收太多，则接收方会告诉发送方要控制发送的速度
    • 复用分用
      • 复用：多个应用进程可同时使用下面的运输层的服务
      • 分用：运输层把收到的信息分别交付给上面应用层中相应的进程

    # 网络层

    • 主要任务是把分组从源端传到目的端，为分组交换网上的不同主机提供通信服务。
    • 网络层传输单位是数据报
    • 补充：数据报过长的时候，就可以把数据报切分成一个又一个小的分组

    功能如下：

    • 功能一：路由选择
      • A电脑---很多路由器---B电脑，如果A电脑要发送数据给B，会经过很多路由，因此也有很多路径选择，则会选择最佳路径。
    • 功能二：流量控制
      • 如果接收方来不及接收太多，则接收方会告诉发送方要控制发送的速度
    • 功能三：差错控制
      • 发送方和接收方会约定好一些规则，当数据到达接受方，接受方会根据这些规则检查分组有没有错，能纠错就纠错，不能纠错就把分组扔掉，这样就能确保传输层所提交的数据都是没有问题的
    • 功能四：拥塞控制
      • 如果所有结点（包括主机和中间系统）来不及接收分组，而要丢弃大量分组的话，网络就处于拥塞状态，因此要采取一定措施，缓解这种拥塞。

    # 数据链路层

    • 主要任务是把网络层传下来的数据报组装成帧
    • 数据链路层的传输单位是帧

    有如下功能：

    • 功能一：

      • 成帧（定义帧的开始和结束）
        • 组装成帧之后，会形成比较长的比特流序列（...0101010101...），即在哪里是帧的开头，哪里是帧的结尾

    • 功能二：差错控制，包括帧错和位错

      • 数据链路层都会有相应的检错或者纠错的方法，如果有差错，数据链路层可能会丢弃该有差错的帧，以免在该网络下继续地传下来，会浪费资源，如果要进行纠错的话，就会用一些可靠的传输协议来进行纠错

    • 功能三：流量控制，同上

    • 功能四：访问（接入）控制

      • 控制对信道的访问，比如有多台主机同时连接在同一条介质上（就像总线型一样），假如在广播式网络中，数据链路层就要控制一下哪台主机可以占用当前这条信道（因为在广播式网络中，同一时间只能有一个人在发送信息，其他都是处于监听状态）

    # 物理层（傻瓜层）

    只需要将比特流转成电信号，然后放到链路上进行传输就ok了

    • 主要任务是在物理媒体（比如同轴电缆，双绞线）上实现比特流的透明传输
    • 物理层传输单位是比特
    • 啥事透明传输：指不管所传数据是什么样的比特组合，都应当能够在链路上传送，给我啥，我就直接放到链路上

    功能如下：

    • 功能一：定义接口特性

      • 比如说，确定连接电缆的插头一定要有多少个菱角，每条菱角应该怎么样连接

    • 功能二：定义传输模式，有如下三种模式

      • 单工模式：只能够是单方向的，比如说有两个人在通信，那么有一个是发送端，那么另外一个人就是接收端，是确定好了的，不能更改了
      • 半双工模式：双方都可以做发送端或者接收端，但是同一时间还是固定好的，要么就只能是发送方，要么就只能是接受方，就像对讲机一样
      • 双工模式：。。。就比如打电话

    • 功能三：定义传输速率

    • 功能四：比特同步

      • 发送方发一个1，接收端就可以准确无误的接收到一个1

    -功能五：比特编码 -曼彻斯特编码。。。
